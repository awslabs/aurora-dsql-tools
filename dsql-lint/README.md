# dsql-lint

Lint SQL files for [Amazon Aurora DSQL](https://aws.amazon.com/rds/aurora/dsql/) compatibility.

Parses SQL and reports errors (unsupported syntax) and warnings (best practice violations) with suggested fixes.

> **Note:** This is an early release covering the most commonly encountered compatibility issues. Additional rules and broader SQL coverage will be added in upcoming releases.

## Installation

```bash
cargo install --path .
```

## Usage

```bash
dsql-lint migration.sql [migration2.sql ...]
```

### Example Output

```
migration.sql:1: ERROR — FOREIGN KEY (REFERENCES) not supported in DSQL (column 'user_id')
  → Remove REFERENCES and implement referential integrity in application layer.
migration.sql:1: ERROR — SERIAL type not supported in DSQL (column 'id')
  → Use: id INTEGER GENERATED BY DEFAULT AS IDENTITY (CACHE 1)
migration.sql:1: WARNING — Table 'settings' missing tenant_id column
  → Add tenant_id VARCHAR(255) NOT NULL for multi-tenant isolation.
```

Exit code `0` if no errors (warnings are OK), `1` if any errors.

## Rules

### Errors (will fail on DSQL)

- FOREIGN KEY / REFERENCES constraints
- CREATE INDEX without ASYNC
- SERIAL / BIGSERIAL / SMALLSERIAL column types
- JSON / JSONB column types
- TRUNCATE statement
- TEMPORARY / TEMP tables
- Array column types
- CREATE TRIGGER
- CREATE EXTENSION
- PARTITION BY
- CREATE FUNCTION / PROCEDURE

### Warnings (best practice)

- CREATE TABLE missing tenant_id column
- ALTER TABLE ADD COLUMN with inline DEFAULT or NOT NULL

## Library Usage

`dsql-lint` is also a library crate:

```rust
use dsql_lint::lint::lint_sql;

let diagnostics = lint_sql("CREATE TABLE t (id SERIAL);");
for d in &diagnostics {
    println!("{}", d.message);
}
```
