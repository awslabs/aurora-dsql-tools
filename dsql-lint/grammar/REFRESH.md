# Refreshing `dsql_grammar.json`

This file is the source of truth for what DSQL accepts. It is generated by
an internal grammar-derivation tool that runs candidate statements against
a real DSQL cluster and accumulates the grammar of what succeeded. Do not
hand-edit.

The `root` field (`"toplevel_stmt"`) is a dispatch rule that isn't itself
defined in this file. The `dsql_lint::grammar::Grammar` loader iterates
[`TOP_LEVEL_RULES`](../src/grammar/mod.rs) instead.

## Regenerating

Replace this file with a fresh JSON produced by the internal grammar
generator (it requires a live DSQL endpoint), then run:

```sh
cargo run --features grammar-diff --release --bin grammar-diff > diff.out
```

Review `diff.out` and open follow-up PRs as the workflow in the dsql-lint
README describes.

## Why it's vendored

A live DSQL endpoint is too much to require for every dsql-lint build.
Committing the artifact keeps refresh a deliberate, reviewable PR; the diff
on this file is the surface-area change.
